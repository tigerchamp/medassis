# 家庭健康助手 - 软件需求规格说明书（修订版）

**版本**：V2.0  
**日期**：2026年7月12日  
**基于原型版本**：家庭健康中心 · 迭代版


## 1. 引言

### 1.1 项目目标
为家庭提供一个集中管理老人病历、用药记录、药品库存的共享工具。核心能力包括：通过拍照和OCR识别自动提取病历、处方、报告、药品信息；支持家庭成员之间的信息共享与权限管理；提供用药提醒、药品过期预警、开药倒计时等主动健康管理功能。系统以“家庭”为管理单元，以“老人”为健康数据主体，兼顾“照护者”与“被照护者”的双重视角。

### 1.2 目标用户
- **老人（被管理者）**：基本信息及健康档案的拥有者，可能不擅长操作智能手机，需要家人代为管理。
- **家人/照护者（核心使用者）**：负责上传、查看、编辑、提醒，是系统的日常操作主体。
- **主管理员（家庭创建者）**：拥有最高权限，可邀请/移除家庭成员、设置权限、管理所有数据。


## 2. 总体功能架构

系统由以下模块组成，以**首页**为信息聚合中心，以**底部导航**为核心操作入口：

| 模块 | 说明 |
|------|------|
| **首页** | 当前选中成员的健康信息聚合页，含用药安排、开药倒计时、药品说明书入口 |
| **病历** | 集中展示病历记录、电子报告、指标趋势 |
| **拍照识别** | 全局统一入口，支持识别病历/处方/报告/药品，结果自动归档 |
| **药箱** | 家庭药品库存管理，含数量/规格、有效期、过期预警 |
| **我的** | 个人设置、家庭成员管理（邀请/授权） |
| **消息中心** | 聚合用药提醒、过期预警、家庭动态等通知 |


## 3. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| **主管理员（创建者）** | 创建家庭组；邀请/移除家庭成员；为所有成员设置编辑权限；管理所有数据。 |
| **家庭成员（授权编辑）** | 查看所有共享信息；上传/编辑本人及被授权老人的健康数据；接收提醒。 |
| **家庭成员（只读）** | 仅查看，不可上传或编辑（由主管理员在“家庭管理”中通过开关控制）。 |

> **权限控制**：在“家庭管理”页面，每位成员旁设有授权开关，开启后该成员可被他人代为管理健康信息（适用于老人本人不会操作的情况）。


## 4. 详细功能需求

### 4.1 用户与家庭管理

#### 4.1.1 成员切换与身份识别
- **左上角家人选择器**：点击下拉菜单显示所有家庭成员列表，选中后**首页内容立即切换**为该成员的健康数据。
- **头部信息展示**：右上角同步显示当前选中的成员姓名及头像，确保用户始终知晓当前查看的是哪位家人的信息。

#### 4.1.2 家庭组管理
- **入口**：位于“我的”页面 → “家庭管理”。
- **成员列表**：展示所有家庭成员及其角色（管理员/成员），每位成员旁设有**授权开关**。
- **授权开关**：开启后，该成员的健康信息允许其他家庭成员代为编辑（适用于老人或不擅操作的家属）。
- **邀请成员**：通过“邀请家人加入”按钮进入邀请页面，生成**二维码**和**邀请码**，并显示**有效期**（默认7天）。
- **接收邀请**：受邀者打开链接后，可选择**是否允许邀请人编辑自己的健康信息**，确认后加入家庭组。

#### 4.1.3 多老人支持
- 同一家庭可管理多位老人（父母、配偶父母等），每位老人拥有独立的健康档案。
- 通过左上角下拉菜单在成员间快速切换查看。

#### 4.1.4 用户数据模型
- **用户表**：ID、姓名、手机号、角色（管理员/成员）、家庭组ID、头像URL、授权状态。
- **家庭组表**：ID、组名、创建者ID、创建时间、邀请码、二维码URL、邀请有效期。


### 4.2 拍照上传与OCR识别（统一入口）

#### 4.2.1 入口设计
- **底部中央圆形按钮**（始终可见）：点击后弹出**扫描类型选择器**，提供四种识别类型：
  - **病历**：扫描纸质病历或电子档，提取就诊信息。
  - **处方**：识别处方药品信息，自动关联用药计划。
  - **报告**：扫描检查报告、化验单，提取指标数据。
  - **药品**：扫描药盒，自动录入药箱并关联过期提醒。
- **药箱页面内部**：也保留“拍照录入药品”快捷入口。

#### 4.2.2 图像处理
- **拍摄/选择**：支持相机拍摄或从相册选择（单次单张，聚焦精准识别）。
- **预处理**：自动裁剪、增强对比度、纠正倾斜，提升OCR识别率。
- **隐私保护**：上传图片默认仅家庭组内可见，可手动选择仅保存识别文本。

#### 4.2.3 OCR识别与校对
- **内容类型识别**：
  - 病历 → 提取就诊日期、诊断、检查指标、医嘱。
  - 处方 → 提取药品名称、用法、剂量、频次、起止时间。
  - 报告 → 提取项目名称、结果数值、参考范围、异常提示。
  - 药品 → 提取药品名称、生产批号、有效期，自动录入药箱。
- **多语言支持**：优先识别印刷体中文/英文；手写体标注“可能不准确”。
- **校对界面**：识别后提供编辑框让用户修改或补充遗漏字段。
- **模拟交互**：识别过程显示加载动画（约1.5秒），完成后弹出结果提示并自动更新对应列表。


### 4.3 自动分析整理

#### 4.3.1 用药管理
- **今日用药安排**（首页核心卡片）：
  - 按早/中/晚分时段展示药品名称、剂量。
  - 每项药品旁设**“待服/已服”切换按钮**，点击后标记服用状态。
  - 显示**下次提醒时间**。
- **开药倒计时**（首页）：
  - 根据剩余药量和每日用量，自动计算**剩余可服用天数**。
  - 显示**建议开药日**，提示用户提前备药。

#### 4.3.2 病历结构化
- **病历记录**（病历页面）：按时间倒序排列所有就诊记录，显示日期、医院、诊断摘要。
- **电子报告**（病历页面）：按时间列出检查报告，支持查看详情和上传新报告。
- **指标趋势**（病历页面）：对定期测量的指标（如血压、血糖）以图表形式展示变化趋势。

#### 4.3.3 药品库存管理
- **家庭药箱**（药箱页面）：
  - 每种药品显示：名称、有效期、**数量与规格**（如“2盒 · 20粒/盒”）。
  - 自动标记“有效”、“即将过期”、“已过期”状态。
  - 过期药品在列表中高亮警示，药箱顶部显示汇总提醒。

#### 4.3.4 药品说明书查询
- 在首页用药安排中，**点击药品名称**打开独立的二级页面，显示该药品的完整说明书（适应症、用法、禁忌等），字体已适配老年人阅读。


### 4.4 家庭共享与协作

#### 4.4.1 共享看板
- 家庭组内所有成员可查看当前选中老人的：
  - 病历时间轴（病历页面）
  - 用药计划表（首页）
  - 药品库存（药箱页面）

#### 4.4.2 授权与协作
- **授权管理**：在“家庭管理”页面，通过成员旁的开关控制是否允许他人代为编辑。
- **数据同步**：家庭数据实时同步，成员可查看彼此的健康信息（在权限范围内）。


### 4.5 提醒与通知

#### 4.5.1 消息中心
- **入口**：顶部消息图标（带未读红点）。
- **消息类型**：
  - 用药提醒（“今晚 21:00 记得服用，剩余5片”）
  - 药品过期预警（“胰岛素注射液已过期，请尽快处理”）
  - 家庭邀请（“王芳 申请加入您的家庭组”）
  - 家庭成员动态（“母亲 · 新检查报告上传”）
- **交互**：点击消息可跳转至对应页面（模拟）。

#### 4.5.2 用药提醒
- 根据用药计划表，在设定时间向家庭成员推送提醒。
- 用户在首页可标记“已服”，记录实际服用时间。
- 漏服超过1小时生成漏服日志并通知管理员。


### 4.6 搜索与导出

#### 4.6.1 搜索（规划中）
- 按关键词（药名、诊断、时间范围）检索所有病历或药品。

#### 4.6.2 数据导出（规划中）
- 一键生成“近期健康小结”（PDF），含病历时间轴+当前用药清单，供就诊时参考。


## 5. 界面与交互设计

### 5.1 页面结构

| 页面 | 核心内容 |
|------|----------|
| **首页** | 左上角家人切换 + 用药安排 + 开药倒计时 |
| **病历** | 病历记录 + 电子报告 + 指标趋势 |
| **拍照** | 底部圆形按钮，弹出扫描类型选择器 |
| **药箱** | 药品列表（含数量/规格）+ 拍照录入入口 |
| **我的** | 个人信息 + 设置 + 家庭管理入口 |
| **消息中心** | 聚合所有通知提醒 |
| **家庭管理** | 成员列表 + 授权开关 + 邀请入口 |
| **邀请页面** | 二维码 + 邀请码 + 有效期 |
| **接收邀请** | 加入确认 + 权限选择 |
| **药品说明书** | 独立二级页面，大字展示 |

### 5.2 适老化设计
- 主界面字体≥16sp，关键按钮高对比度。
- 药品说明书独立展示，字号可放大。
- 操作路径短：拍照→选类型→识别→自动归档，三步完成。
- 左上角家人切换采用大按钮，易于点击。


## 6. 非功能需求

### 6.1 性能要求
- OCR识别单张图片平均耗时 ≤ 3秒（网络良好时）。
- 上传图片最大单张 ≤ 20MB，支持JPG/PNG/HEIC。
- 通知推送延迟 ≤ 10秒。
- 页面切换动画流畅，加载时间 ≤ 500ms。

### 6.2 安全性
- **数据加密**：所有健康数据在传输和存储时采用AES-256加密。
- **访问控制**：严格校验Token，禁止非家庭成员访问。
- **数据归属**：用户可随时导出全部数据并删除账户，彻底清除云端记录。

### 6.3 可用性与易用性
- **适老化设计**：主界面字体可调大（≥16sp），关键按钮使用高对比度颜色。
- **操作路径短**：拍照→选择类型→识别→保存，四步完成一条记录。
- **离线模式**：无网络时可本地拍照并暂存，联网后自动上传识别。
- **底部栏常驻**：所有页面底部导航始终可见，确保快速切换。

### 6.4 可扩展性
- 预留与主流健康硬件（如智能血糖仪、血压计）的数据接口（蓝牙/API），后续可接入自动同步数据。

### 7.2 数据存储
- 图片存储于云对象存储（如OSS），文本数据存于关系数据库。
- 保留所有历史版本（如修改过的识别结果），支持时间轴回看。


## 8. 接口需求

- **OCR服务接口**：调用成熟第三方OCR API（如百度、腾讯、阿里云医疗OCR），优先支持医疗单据识别模型，返回结构化数据供校对。
- **推送服务**：接入极光推送或各厂商通道（华为、小米、苹果APNs）。
- **图片上传接口**：支持JPG/PNG/HEIC格式，单张≤20MB，自动压缩与预处理。


## 9. 约束与假设

- **初期版本范围**：聚焦拍照识别、家庭共享与用药管理，暂不做复杂临床决策支持。
- **手写体识别准确率不保证**：建议用户手动校对。
- **隐私合规**：需符合《个人信息保护法》，在首次使用时明示获取相册、相机、通知权限的目的。
- **原型参考**：当前设计以移动端为主（420px宽度），后续可适配平板和Web端。
